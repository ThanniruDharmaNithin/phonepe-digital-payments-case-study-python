## 📌 Project Overview

This project analyzes **PhonePe digital payment transaction data across India** along with demographic information at the **state and district levels**.

The analysis explores how **digital payments are growing across regions**, how **device usage affects adoption**, and how **demographic characteristics influence transaction behavior**.

Using Python data analysis libraries, the study performs **data exploration, validation, statistical analysis, and visualization** to uncover meaningful insights about the **digital payments ecosystem in India**.

# ✔ Task 3: Data Quality Checks

Data validation was performed by comparing **district-level data with state-level aggregates**.

### Validation Objectives

* Verify district transaction totals match state totals
* Identify inconsistencies in transaction amounts
* Validate registered user counts

### Importance

Such validation is important in real-world analytics because data often originates from **multiple sources and systems**.

If discrepancies exist, they may indicate:

* Missing district records
* Aggregation inconsistencies
* Data synchronization issues

# 🌍 Demographic and Transaction Correlation

The analysis indicates:

* Districts with higher population density tend to have higher transaction activity.
* Urban districts demonstrate stronger digital payment adoption.
* Rural districts may experience lower adoption due to infrastructure limitations.

This confirms that **demographic characteristics influence digital payment behavior**.
